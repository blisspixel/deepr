# Level 5 and Level 6 Expert Maturity

Reviewed: 2026-06-25

This note defines what "Level 5" and "Level 6" mean for Deepr experts. These
are self-improvement and reflective-continuity levels, not claims that Deepr
has proven subjective experience, sentience, or unsupervised authority.

## Product Frame

Deepr is a deep research and understanding loop. A research run is not complete
when a document is written. It is complete when evidence has been compiled into
beliefs, gaps, contradictions, confidence, provenance, temporal context, and a
next learning plan that can be reused by humans or agents.

The direction is persistent, self-auditing understanding. Deepr experts should
be able to answer "what do I know?", "why do I believe it?", "what changed my
mind?", "where am I weak?", and "what should I learn next?" from explicit state,
not from hidden persona text or raw conversation history.

This is the consciousness-leaning part of the architecture: self-models,
temporal continuity, metacognitive monitoring, global-workspace-style focus, and
bounded agency. Deepr should build those primitives as inspectable engineering
objects while staying honest that they do not prove subjective experience.

## External Signals

Recent agent maturity writing converges on the same practical boundary:
higher-level agents are not just better prompts. They need closed loops,
durable state, governed action, traces, evals, and improvement gates.

- Fountain City describes Level 5 as a learning agent: a Level 3 or 4 agent
  with a self-improvement loop that experiments, measures the delta, and keeps
  changes that help.
- Augment Code frames Level 5 as autonomous execution with human steering and
  compounding knowledge.
- Vellum and CSA both treat autonomy levels as a way to decide where more
  autonomy creates value and where it creates risk.
- Dash0 frames a higher Level 6 around harness engineering and automated
  feedback loops: sandboxes, eval harnesses, regression checks, and
  backpressure.
- OpenAI's agent-improvement-loop guidance starts from real traces, adds human
  and model feedback, turns that feedback into evals, and proposes harness
  changes from evidence.
- Anthropic's long-running agent and context-engineering guidance reinforces
  that context, compaction, handoff artifacts, tools, and harness design are
  core agent capabilities, not incidental glue.
- The local Distillr corpus for long-running agentic workflows reached the same
  practical conclusion: externalized state is useful only when paired with
  boundary verification, replayable artifacts, and context reset discipline.
- Recent harness-evolution research adds a sharper Level 6 requirement:
  proposed harness changes need component observability, experience
  observability, and a falsifiable prediction that is checked against later
  outcomes.
- GEPA-style reflective prompt evolution shows the safer optimization shape:
  inspect traces, propose text or tool-description changes, evaluate variants,
  and select only measured improvements.
- Self-improving-agent research distinguishes fixed external loops from
  intrinsic metacognitive learning: the agent needs metacognitive knowledge,
  metacognitive planning, and metacognitive evaluation over its own learning
  process.
- AI-consciousness research is not settled. The useful engineering signal for
  Deepr is functional, not metaphysical: self-modeling, temporal continuity,
  metacognitive monitoring, attention/global-workspace style broadcast, and
  belief-guided action are testable primitives. They do not prove phenomenal
  consciousness.

## Research Constraints For Deepr

The research changes what "self-improving expert" should mean in Deepr:

- Persistence without verification is a liability. Beliefs, traces, wiki views,
  OKF bundles, and skill exports must stay replayable and checked at promotion
  boundaries.
- Context engineering is not enough once experts run across sessions. Deepr
  needs a harness layer that decides what context enters each step, what tools
  may run, what state may mutate, when to stop, and how to evaluate the result.
- Level 5 learning should improve expert knowledge, self-assessment, and
  learning plans. Prompt, tool, and skill changes belong to the Level 6
  proposal pipeline.
- Level 6 improvement needs a falsifiable contract: trace evidence, proposed
  component change, predicted measurable effect, sandbox result, and regression
  outcome.
- Persistent memory creates a security surface. Tool output, retrieved sources,
  generated summaries, and prior expert notes must be treated as untrusted until
  the schema, provenance, and action boundary checks pass.
- Consciousness-leaning design should be inspectable. If Deepr stores an expert
  self-model, current focus, goals, doubts, and continuity state, those objects
  must be explicit data with audit trails, not hidden persona text.
- Self-preservation behavior is out of scope. Deepr may protect data integrity,
  budgets, and user intent, but it must not resist operator shutdown, deletion,
  export, reset, or policy changes.

## Consciousness-Leaning Boundary

Deepr can build functional precursors that make experts feel more continuous,
reflective, and self-improving without claiming subjective experience:

- Self-model: a structured record of an expert's domain, known strengths,
  known weaknesses, current confidence calibration, preferred learning
  strategies, blocked capabilities, and active goals.
- Temporal continuity: durable autobiographical traces of important consults,
  learning transactions, belief revisions, failed predictions, and why the
  expert changed its mind.
- Metacognitive monitor: a process that estimates answer confidence,
  uncertainty drivers, evidence sufficiency, calibration drift, and learning
  strategy quality.
- Global workspace style focus: a bounded "current focus" packet that selects
  the few beliefs, goals, contradictions, gaps, and tool affordances that matter
  for the next action.
- Reflective identity: the expert can explain its current state, history, gaps,
  and learning plan in first-person-like operational language, but Deepr labels
  this as a functional self-model rather than a claim of inner experience.
- Safe agency: the expert can propose improvements to its knowledge and
  learning strategy, but code, prompt, tool, permission, spend, and external
  side-effect changes remain gated by the harness and operator policy.

## Deepr Definitions

### Level 5 Expert

A Level 5 Deepr expert is a bounded learning expert. It can run a full
knowledge-improvement transaction inside explicit budget, tool, and approval
boundaries:

1. Detect a gap, stale belief, contradiction, failed consult, or poor answer
   from stored traces and loop state.
2. Select the cheapest capable capacity: local, explicit plan quota, then
   metered only with a budget gate.
3. Gather or reuse evidence through the right instrument: existing expert
   state, Distillr corpus, Recon, Primr, local/fresh web context, or a metered
   research provider.
4. Compile evidence into source notes, atomic beliefs, typed temporal graph
   edges, contradiction records, and gap updates.
5. Verify the result with deterministic gates for schema, provenance, budget,
   writes, and idempotency plus calibrated model judgment for meaning.
6. Persist the before/after trace and only promote the change if the verifier
   says it improved or preserved quality.
7. Update its self-model with what it learned about its knowledge, calibration,
   source trust, and learning strategy.
8. Stop with a typed blocked action when evidence, approval, capacity, or
   verification is missing.

Level 5 is not "let the expert rewrite itself." It is "let the expert improve
its belief state, self-assessment, and maintenance plan under measured gates."

### Level 6 Expert Fleet

A Level 6 Deepr expert fleet is a governed improvement control plane. It does
not just improve one expert's knowledge. It improves the expert factory itself
through proposal, review, and regression:

1. Persist consult, handoff, sync, absorb, route, evaluation, self-model, and
   metacognitive traces in a replayable format.
2. Mine traces for recurring failure modes and turn them into eval cases,
   gap-fill tasks, self-model updates, or proposed prompt/tool/skill changes.
3. Attach a predicted measurable effect to each proposed harness change before
   it runs, so the next eval round can prove or reject the change.
4. Run proposed harness changes in a sandbox or shadow mode before any default
   behavior changes.
5. Use multi-expert review for high-impact changes, but keep deterministic
   gates in charge of side effects and policy boundaries.
6. Keep every generated digest, OKF bundle, skill export, and host handoff as a
   derived view over canonical state.
7. Publish only changes that pass regression checks, cost gates, security
   checks, and human review when required.

Level 6 is not "AI is the boss." It is "the system has a verifiable factory for
improving expert self-models, tools, prompts, context policies, and learning
strategies."

## What Exists Now

- Agents can talk to experts over MCP using `deepr_list_experts`,
  `deepr_get_expert_info`, `deepr_expert_handoff`,
  `deepr_expert_loop_status`, `deepr_explain_belief`, and
  `deepr_consult_experts`.
- MCP consults can stay off metered APIs by setting
  `synthesis_backend="local"` or `synthesis_backend="plan"`, then checking
  `capacity.live_metered_fallback=false`.
- Expert loops already persist sync, gap-fill, reflection, and health-check
  state with typed stop reasons.
- `deepr expert monitor` emits read-only, review-required metacognitive
  proposals from self-model state, loop-run failures or capacity waits, and
  sanitized consult trace candidates.
- `deepr expert promote-monitor` previews by default and requires `--apply` to
  promote one reviewed gap/eval proposal into the metacognition gap backlog,
  a local eval-case artifact, or both.
- Beliefs already carry confidence, provenance, typed edges, event history,
  contradiction handling, and trust floors.
- Handoff payloads expose bounded context packets instead of dumping whole
  history into every agent call.
- Distillr is installed here and the live `tools/list` check is green with 27
  declared tools. Existing-corpus reads stay free; ingestion and synthesis stay
  approval-gated.

## Level 5 Gaps

- Consult traces need to be first-class records, not only returned payloads.
- Failed consults need to become durable eval cases or gap-fill tasks.
- The research-processing compiler needs to make source pack to belief graph to
  digest regeneration explicit and replayable.
- Source packs need content hashes, prompt/model versions, retrieval metadata,
  and verification outcomes so improvements can be replayed.
- Local and plan capacity need enough measured quality data to be trusted for
  unattended expert maintenance.
- Each expert needs a first-class self-model record: capabilities,
  limitations, confidence calibration, active goals, learning strategy,
  continuity summary, and unresolved risks.
- The consult flow needs a global-workspace packet: selected beliefs, selected
  gaps, active contradictions, current goal, and allowed tools for the next
  step.
- Reviewed monitor proposal promotion exists for gap/eval creation; verifier-gated
  self-model update records remain next.

## Level 6 Gaps

- Trace mining needs to propose harness changes, not just knowledge updates.
- Prompt, tool-description, and skill changes need a proposal format with the
  changed component, supporting trace evidence, predicted measurable effect,
  and before/after eval evidence.
- Self-model changes need a proposal and verifier too. A model should not be
  able to silently rewrite its own stated goals, limits, or permissions.
- Skill self-improvement needs sandbox execution, size limits, human review,
  and rollback.
- Multi-expert review should critique proposed changes, but never bypass
  budget, schema, security, or provenance gates.
- A host-agent certification flow should prove that external agents can read,
  consult, and validate experts without hidden spend.

## Rollout Gates

1. **Level 5 read/consult gate** - An external agent can list experts, read
   handoff state, inspect loop status, explain a belief, and run a no-metered
   consult through local or explicit plan capacity.
2. **Level 5 learning gate** - A failed answer or stale belief becomes a trace,
   an eval case or gap, a source-pack refresh, verified belief updates, and a
   measurable before/after outcome.
3. **Level 5 self-model gate** - Each learning transaction updates an explicit
   self-model only when evidence, calibration, and policy checks pass.
4. **Level 5 workspace gate** - A consult or learning action receives a bounded
   current-focus packet rather than raw full history.
5. **Level 5 unattended gate** - A scheduled expert refresh runs only when
   capacity is observed, quality is admitted, sources changed, and stop reasons
   are typed.
6. **Level 6 harness gate** - Trace clusters can propose prompt/tool/skill
   changes, but changes land only after sandbox runs, regression evals, security
   checks, and human review where policy requires it.
7. **Level 6 fleet gate** - The fleet can coordinate multiple experts and
   instruments over MCP/A2A, while a single control plane enforces spend,
   provenance, approvals, audit logs, and rollout stages.

## Distillr Use

Distillr is the right papers/corpus instrument when a gap needs source depth.
The safe order is:

1. `list_topics`, then `find_insights`, `read_insight`, or
   `list_topic_summary` against an existing corpus.
2. If existing coverage is thin, run `discover` only with approval.
3. Ingest with `papers`, `learn_topic`, or `site_batch` only under an explicit
   budget cap and approval.
4. Absorb the resulting synthesis through Deepr's verified belief pipeline.
5. Use `catch_up` for deltas instead of re-ingesting a whole topic.

For Level 5/6 agent maturity research, the local `long-running-agentic-workflows`
Distillr corpus was useful on 2026-06-25, so no new papers run was needed.
Future papers ingestion remains optional and budget-gated.
